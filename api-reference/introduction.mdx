Welcome to the Allegory Funders API documentation. This API provides:

- **Funder Search**: Fast, edge-cached access to nonprofit funder information from IRS 990 filings
- **Grant History**: Historical grant-making data to understand funding patterns
- **Funding Opportunities**: Organization-scoped opportunity tracking and management

## Base URL

All API requests should be made to:

```
https://allegory-middleware.ray-392.workers.dev
```

## Authentication

<Warning>
All API requests require authentication via an API key.
</Warning>

Include your API key in the request header:

```
X-API-Key: ak_xxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### Getting an API Key

Contact us at **hello@allegory.world** to request your API key. Keys are issued in the format `ak_xxxxx...`.

<Tip>
Never expose your API key in client-side code or commit it to version control. Store it securely as an environment variable.
</Tip>

## Rate Limits

The API has a rate limit of **100 requests per minute** per API key.

## Data Freshness

Data is synced daily from our analytics warehouse at **4am UTC**. New IRS 990 filings are typically available within 24-48 hours of being filed.

## Response Format

All API responses follow a consistent format:

<ResponseField name="success" type="boolean">
  Indicates whether the request was successful
</ResponseField>

<ResponseField name="data" type="object | array">
  The response data. Structure varies by endpoint.
</ResponseField>

<ResponseField name="error" type="string">
  Error message (only present when success is false)
</ResponseField>

## Getting Started

<Steps>
<Step title="Get your API key">
  Contact hello@allegory.world to request your API key.
</Step>

<Step title="Explore the API endpoints">
  Browse the endpoints in the sidebar to see available operations for searching funders and accessing grant history.
</Step>

<Step title="Try the API playground">
  Use the interactive API playground on each endpoint page to test requests directly from your browser with your API key.
</Step>

<Step title="Review the data models">
  Check the response schemas to understand the structure of returned data.
</Step>
</Steps>

## Quick Example

Here's a simple example to get you started:

```bash
curl -X GET 'https://allegory-middleware.ray-392.workers.dev/v1/funders?limit=5' \
  -H 'X-API-Key: YOUR_API_KEY' \
  -H 'Accept: application/json'
```
