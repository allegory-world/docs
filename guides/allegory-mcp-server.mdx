The Allegory MCP server exposes all 15 public API endpoints as [Model Context Protocol](https://modelcontextprotocol.io/) tools, enabling LLM agents to query funder data and manage funding opportunities directly.

## What is MCP?

The Model Context Protocol (MCP) is an open protocol that enables AI assistants to connect to external data sources and tools. By connecting to the Allegory MCP server, your AI assistant can:

- Search for nonprofit funders by name, location, or focus area
- Retrieve detailed funder information and identifiers
- Query grant history and funding patterns
- Look up and manage funding opportunities
- Access funder contacts and people data

## Prerequisites

- An Allegory API key (contact [hello@allegory.world](mailto:hello@allegory.world))
- An MCP-compatible client (Claude Desktop, Cursor, or MCP Inspector)

## Connect from Claude Desktop

Add the following to your Claude Desktop config file (`claude_desktop_config.json`):

```json
{
  "mcpServers": {
    "allegory": {
      "url": "https://api.allegory.world/mcp",
      "headers": {
        "X-API-Key": "your-key-here"
      }
    }
  }
}
```

<Warning>
Replace `your-key-here` with your actual API key. Never share your API key publicly.
</Warning>

## Connect from Cursor

Add the following to your Cursor MCP settings (`.cursor/mcp.json` in your project root):

```json
{
  "mcpServers": {
    "allegory": {
      "url": "https://api.allegory.world/mcp",
      "headers": {
        "X-API-Key": "your-key-here"
      }
    }
  }
}
```

## Test with MCP Inspector

You can test the MCP server using the official MCP Inspector:

```bash
npx @modelcontextprotocol/inspector@latest
```

Enter `https://api.allegory.world/mcp` as the server URL, add the `X-API-Key` header with your API key, and click **Connect**.

## Authentication

The MCP server uses the same `X-API-Key` authentication as the Allegory API. Include your API key in the headers on every MCP request.

```
X-API-Key: your-key-here
```

## Available tools

The MCP server provides 15 tools split across two categories: read-only funder tools and organization-scoped opportunity tools.

### Funder tools (read-only)

These tools query the funder database and grant history. They are read-only and do not modify any data.

| Tool | Description |
|------|-------------|
| `allegory_search_funders` | Search funders by name, country, type, focus areas |
| `allegory_get_funder` | Get funder by EIN |
| `allegory_get_funder_by_id` | Get funder by UUID (includes identifiers) |
| `allegory_get_funder_by_identifier` | Look up by external identifier method/value |
| `allegory_get_funder_history` | Grant history by EIN |
| `allegory_get_funder_history_by_id` | Grant history by UUID (with filters) |
| `allegory_get_funder_contacts` | Contacts by EIN |
| `allegory_get_funder_contacts_by_id` | Contacts by UUID |
| `allegory_search_grant_history` | Cross-funder grant history search |

### Opportunity tools (org-scoped)

These tools manage funding opportunities scoped to your organization. Write operations (create, update) modify data in your org's opportunity tracker.

| Tool | Description |
|------|-------------|
| `allegory_list_opportunities` | List opportunities with filters |
| `allegory_get_opportunity` | Get opportunity by ID |
| `allegory_lookup_opportunity` | Look up by Donor + program_name |
| `allegory_create_opportunity` | Create a new opportunity |
| `allegory_update_opportunity` | Update opportunity by ID |
| `allegory_update_opportunity_by_lookup` | Update by Donor + program_name |

## Example prompts

Once connected, you can ask your AI assistant questions like:

### Research and analysis
- "Which foundations focus on climate change in California?"
- "What's the average grant size for education-focused funders?"
- "Show me funders that support both arts and education"

### Grant writing
- "Find funders that have given to similar organizations"
- "What's the typical grant range for the Ford Foundation?"
- "Show me the Gates Foundation's giving history for the last 3 years"

### Opportunity management
- "Create a new funding opportunity for the Ford Foundation's climate grant program"
- "List all my open funding opportunities"
- "Update the status of our Gates Foundation application to in_progress"

### Due diligence
- "Is this foundation verified in Allegory?"
- "What's their giving history over the past 5 years?"
- "Who are the key contacts at the MacArthur Foundation?"

## Rate limits

The MCP server respects the same rate limits as the Allegory API: **100 requests per minute** per API key.

## Data sources

All data comes from the Allegory API, which aggregates information from:
- IRS 990 and 990-PF filings
- Live web search
- Public and private APIs
- AI-enriched contact data
